# ADR-003: Scope 3 계산 엔진 설계 및 MVP 카테고리 선정

- **상태**: Accepted
- **날짜**: 2026-05-18
- **결정자**: esg-t2 기획팀

---

## 배경

GHG Protocol은 15개(+신설 Category 16)의 Scope 3 카테고리를 정의한다. 전 카테고리를 MVP에 구현하는 것은 과도하다. 반면, 규제(GHG Protocol Phase 1 Update, 2026-03)는 Scope 3 배출 출처의 95%를 수량화하고, 모든 제외 이유를 공개하도록 요구한다.

---

## 결정

### MVP 구현 카테고리: Category 1·2·11

**선정 근거**:

| Category | 이름 | 선정 이유 |
|---|---|---|
| Cat.1 | 구매재화·서비스 | 대부분 기업의 최대 배출원 (50~80%); KSSB 2 주요 공시 항목 |
| Cat.2 | 자본재 | 제조업 기업의 주요 배출원; 데이터 확보 상대적 용이 |
| Cat.11 | 판매제품 사용 | 소비재·IT 기업의 최대 배출원; 제품 설계 개선 인센티브 |

**나머지 카테고리**: 95% 커버리지 보고서에서 "제외 이유" 자동 문서화로 GHG Protocol 요건 충족.

### Category 16 (Facilitated Emissions) 처리

GHG Protocol Phase 1 Update(2026-03)에서 신설 예고. 최종 기준 2027년 말 확정 예상. 전략:
- DB 스키마에 `scope3_category = 16` 허용 (CHECK 제약: 1~16)
- 계산 엔진은 최종 기준 확정 후 구현
- 설계 문서에 Category 16 계산 공식 자리 표시 (TODO 주석)

### 데이터 품질 점수

GHG Protocol이 지출 기반 → 공급업체 특정 데이터로의 전환을 요구하므로, 모든 Scope 3 레코드에 데이터 품질 수준을 기록한다:

```
SPEND_BASED       → 낮음 (지출액 × 산업 평균 계수)
AVERAGE_DATA      → 중간 (산업 평균 활동 데이터)
SUPPLIER_SPECIFIC → 높음 (공급업체 실제 측정 데이터)
```

## 대안 검토

| 대안 | 장점 | 단점 | 탈락 이유 |
|---|---|---|---|
| 전체 15개 카테고리 MVP 구현 | 완전성 | 개발 기간 3배 이상, 데이터 수집 불가 카테고리 존재 | 현실적 불가 |
| Cat.1만 구현 | 단순 | 95% 임계값 달성 어려움, M+1 확장 범위 큼 | 규제 부적합 |
| Cat.1·2·11 구현 + 95% 보고서 | 규제 충족, 현실적 범위 | Cat.3~15 계산 미구현 | **선택** |

---

## 결과

- MVP에서 대부분 기업의 Scope 3 중요 배출원 커버
- 95% 커버리지 보고서로 GHG Protocol 규제 요건 충족
- Cat.3~15는 M+1에서 순차 추가 가능한 플러그인 구조로 설계
- Category 16은 데이터 모델 준비 → 리스크 최소화
